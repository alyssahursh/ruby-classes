Ruby Classes, Explained with Outfits

Struggling to understand Ruby classes and instances? Let me try to explain it with something you do every day (whether you like it or not): create an outfit.

To make the idea of outfits fit the idea of Ruby Classes, let's pretend you have a little bit of a style uniform: Every day, your outfit consists of a top, a bottom, a hijab, a pair of shoes, and a piece of jewelry.

To make a new outfit, you need to specify five items.

Here's a helpful drawing:
And here's some Ruby code: 
class Outfit
  attr_accessor :top, :bottom, :hijab, :shoes, :jewelry

  def initialize(top, bottom, hijab, shoes, jewelry)
    @top = top
    @bottom = bottom
    @hijab = hijab
    @shoes = shoes
    @jewelery = jewelery
  end

end

This Ruby class called "Outfit" is just the code that tells you what an outfit is made of. It says: when you want to make a new outfit, you'll need to specify a top, bottom, hijab, shoes, and jewelry.

So, let's get dressed.

On Monday, you put together a new outfit, and you name it monday_outfit.

Here's a helpful drawing:
And here's some Ruby code: 
monday_outfit = Outfit.new(
  "blue polka dot shirt",
  "navy jeans",
  "teal scarf",
  "black pointy flats",
  "sapphire earrings")

This code says: Make a new instance of the class "Outfit," call it "monday_outfit" and hold on to the details about the top, bottom, hijab, shoes, and jewelry.

Great, but why is this useful?

Now that we have all these instances, we can get information from them.

Like, hey, what bottom did I wear on Tuesday? 

puts tuesday_outfit.bottom

returns 

peach skirt



Or, which hijab did I wear on Friday? 

puts friday_outfit.hijab

returns 

pink scarf



We can even update an outfit: 

wednesday_outfit.shoes = "navy converse"
puts wednesday_outfit.shoes

returns 

navy converse



Or test if an aspect of an outfit matches a particular string: 

puts monday_outfit.top = "blue polka dot shirt"

returns 

true
